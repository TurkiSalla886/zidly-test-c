# Zidly TEST C — تعليمات التشغيل

> **الهدف:** التحقق من جدوى المعمارية الهجينة (iframe + external) قبل كتابة كود الإنتاج.
>
> **المدة:** 30 دقيقة (10 دقائق نشر + 5 دقائق تكوين Salla + 15 دقيقة فحص نتائج)

---

## ما يفحصه هذا الاختبار (8 جوانب)

1. هل الصفحة تعمل داخل iframe؟
2. ما هو origin الصفحة الأم؟
3. هل تحميل `@salla.sa/embedded-sdk` نجح؟
4. ما هي الدوال (methods) المتاحة في SDK؟
5. هل تهيئة SDK نجحت؟
6. ما هي الرسائل التي ترسلها سلة عبر postMessage؟
7. زمن الاستجابة الشبكي (Latency)؟
8. هل CSP يسمح بـ scripts خارجية؟

النتائج تظهر تلقائياً بألوان (✅ أخضر / ⚠️ أصفر / ❌ أحمر) وملخص نهائي.

## الخطوة 1: نشر الصفحة (10 دقائق)

عندك 3 خيارات. **أنصح بـ Vercel** لأن حسابك جاهز:

### الخيار أ — Vercel (الموصى به)

#### الطريقة الأسرع (Drag & Drop):

1. ادخل [vercel.com/new](https://vercel.com/new)
2. **اسحب ملف `index.html`** من مجلد `zidly-test-c` على لوحة Vercel
3. أعطه اسم مشروع: `zidly-test-c`
4. اضغط **Deploy**
5. خلال 10 ثوانٍ ستحصل على رابط مثل:
   ```
   https://zidly-test-c.vercel.app
   ```

**انسخ هذا الرابط — ستحتاجه في الخطوة 2.**

#### بديل (Vercel CLI لو تفضّل terminal):
```bash
cd zidly-test-c
npx vercel --yes
```

### الخيار ب — GitHub Pages (لو واجهت مشكلة في Vercel)

1. أنشئ repo جديد على GitHub اسمه `zidly-test-c` (Public)
2. ارفع `index.html` فقط
3. Settings → Pages → Source: `main` branch, root folder
4. خلال دقيقة، الرابط يكون:
   ```
   https://[username].github.io/zidly-test-c/
   ```

### الخيار ج — Cloudflare Pages

1. ادخل [pages.cloudflare.com](https://pages.cloudflare.com)
2. Create project → Direct upload
3. اسحب `index.html`
4. تحصل على رابط `https://zidly-test-c.pages.dev`

## الخطوة 2: اختبار أولي (قبل تكوين Salla)

**قبل الذهاب لسلة**، افتح الرابط في متصفحك مباشرة لتتأكد أنه يعمل:

```
https://zidly-test-c.vercel.app
```

**ما يجب أن تراه:**
- صفحة عربية بعنوان "🧪 Zidly TEST C"
- 8 بطاقات اختبار تظهر نتائج بألوان مختلفة
- ملخص في الأسفل

**النتائج المتوقعة في هذا الفتح المباشر (ليست داخل سلة):**

| الاختبار | النتيجة المتوقعة |
|---|---|
| 1. iframe detection | ℹ️ "نعمل في تبويب مستقل" (هذا متوقع، ليس فشلاً) |
| 2. Parent origin | ℹ️ "تبويب مستقل" |
| 3. SDK loaded | ✅ يجب أن ينجح |
| 4. SDK methods | ✅ يجب أن يكتشف الـ API |
| 5. SDK init | قد ينجح أو يفشل (طبيعي خارج iframe) |
| 6. postMessage | ℹ️ "لم تُستلم رسائل" (طبيعي) |
| 7. Latency | ✅ يجب أن يقيس الزمن |
| 8. CSP | ✅ يجب أن ينجح |

**إذا فشل اختبار 3 (SDK لم يُحمَّل):** هناك مشكلة في الإنترنت أو ad blocker. حلّ المشكلة قبل المتابعة.


## ❓ مشاكل محتملة وحلولها

### المشكلة: الصفحة لا تفتح في iframe، فقط تظهر فارغة

**السبب:** سلة قد تطلب أن يكون App URL على HTTPS (وليس HTTP).

**الحل:** تأكد أن رابط Vercel يبدأ بـ `https://`.

### المشكلة: "X-Frame-Options" خطأ في Console

**السبب:** Vercel أحياناً يضيف header يمنع iframe.

**الحل:** أضف ملف `vercel.json` بهذا المحتوى في نفس مجلد `index.html`، ثم أعد النشر:

```json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "X-Frame-Options", "value": "ALLOWALL" }
      ]
    }
  ]
}
```

### المشكلة: لا أرى تغييراً بعد تعديل App URL

**السبب:** سلة قد تخزّن App URL في cache.

**الحل:**
1. أزل التطبيق من المتجر التجريبي
2. أعد تثبيته
3. الآن سيستخدم الرابط الجديد

### المشكلة: Vercel يطلب login

**الحل:** سجّل دخول عبر GitHub (نفس الحساب المستخدم في خطة Zidly).
